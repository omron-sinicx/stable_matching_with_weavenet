# stable_matching_val-test

Validation and test datasets for [omron-sinicx/weavenet-egal](https://github.com/omron-sinicx/weavenet-egal).

## Provenance

For each `(distribution, agent count, split)` combination this archive
contains 1000 NPZ samples. The samples come from one of two sources:

1. **Paper-authentic** (40 combinations): bit-exact NPZs from the paper
   authors' submission tarball. These match the data used to produce
   the numbers in the original paper.
2. **Paper-seed-regenerated** (40 combinations): generated by
   `scripts/generate_valtest_data.py --paper-seeds` in the repo, which
   re-seeds numpy with the paper's defaults (135789 for validation, 2456
   for test) once per (split, dist, size) script invocation, then lets
   numpy state evolve through the 1000-sample loop. This mirrors the
   paper's `generate_dataset.py` behavior. Used for combinations that
   were not present in the upload from the authors (mainly N=100 and
   most non-N=20,30 entries for distributions other than UU).

The split between (1) and (2) is documented in
`docs/papers/legacy_code/datasets/` in the source repo.

## Layout

```
stable_matching_val-test/
├── validation/
│   ├── UU3x3.txt          # list of NPZ paths, one per line
│   ├── UU3x3/
│   │   ├── 00000.npz
│   │   ├── 00001.npz
│   │   ...
│   ...
└── test/
    └── (similarly)
```

The repo's `configs/datamodule/val_test_data/*.yaml` expect this layout
under `data/shared/{validation,test}/`.

## NPZ schema (per file)

| key             | shape          | dtype   | meaning                              |
|-----------------|----------------|---------|--------------------------------------|
| `sab`           | `(N, M)`       | float32 | side-a satisfaction in `[0.1, 1.0]`  |
| `sba`           | `(M, N)`       | float32 | side-b satisfaction in `[0.1, 1.0]`  |
| `matches`       | `(K, N, M)`    | float32 | K stable matchings (binary)          |
| `fairness`      | `(K,)`         | float32 | per-match balance cost               |
| `satisfaction`  | `(K,)`         | float32 | per-match total satisfaction         |
| `gs_matches`    | `(2, N, M)`    | float32 | GS a-optimal then b-optimal          |
| `SexEqualityCost`| scalar (or 1,) | float32 | min over `matches`                  |
| `EgalitarianCost`| scalar (or 1,) | float32 | min over `matches`                  |

Paper-authentic NPZs additionally include `cab`, `cba` (rank-cost form
of `sab`, `sba`) and `algorithm` (str). These extra keys are ignored by
the current loader (`src/datamodules/stable_matching.py::NpzDataset`).

For paper-authentic NPZs `K=1` or `K=2`; for paper-seed-regenerated NPZs
`K=2` (GS man-optimal, GS woman-optimal). The loader's `collate_fn`
already handles variable K.

## Regenerating

If you'd like to regenerate this dataset from scratch (e.g., to verify
or to extend to additional N), use the generator script in the source
repo:

```
python scripts/generate_valtest_data.py --paper-seeds --out data/shared
```

This produces statistically equivalent (but not bit-exact) data using
the paper's seed defaults.
